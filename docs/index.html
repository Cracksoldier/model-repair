What it repairs

Every defect that breaks your slicer

model-repair uses CGAL's Polygon Mesh Processing module — the same geometry kernel used in research and production CAD — to detect and fix the six classes of mesh errors that cause failed prints.

🔗

Duplicate vertices

STL files store triangles as independent vertex triples with no sharing. Merging them is required before any topology analysis can proceed.

🗑️

Degenerate triangles

Zero-area faces with collinear or coincident vertices corrupt normal computation and hole detection. Removed before any further processing.

⚠️

Non-manifold geometry

Edges shared by more than two faces, or vertex fans that are not disk-topology. Slicers cannot compute infill on non-manifold meshes.

🔄

Inconsistent normals

Faces whose winding order points inward instead of outward. Causes inside-out or partially inverted prints and incorrect support generation.

🕳️

Holes & open boundaries

Missing faces left by degenerate removal, non-manifold splitting, or upstream CAD export errors. Filled with smooth or flat triangulation.

✂️

Self-intersections

Overlapping triangles from boolean CSG operations or poorly exported models. Removed using CGAL's exact-arithmetic kernel for robustness.

🖥️

3D Before / After preview

The GUI opens a side-by-side OpenGL window after every repair. Rotate, pan, and zoom with the mouse — both views share a synchronized camera so you can compare the meshes without losing orientation.

Getting started

Build from source

model-repair is built with CMake. FetchContent handles lib3mf, CLI11, spdlog, and Catch2 automatically — only system packages are needed upfront.

1 — Install system dependencies

sudo pacman -S cgal eigen3 boost gmp mpfr qt6-base cmake git
sudo apt install libcgal-dev libeigen3-dev libboost-all-dev \
  libgmp-dev libmpfr-dev qt6-base-dev cmake git
sudo dnf install CGAL-devel eigen3-devel boost-devel \
  gmp-devel mpfr-devel qt6-qtbase-devel cmake git

2 — Clone and configure

bash 
git clone https://github.com/Cracksoldier/3d-print-model-repair.git
cd 3d-print-model-repair

# Downloads lib3mf, CLI11, spdlog, Catch2 automatically
cmake --preset release

3 — Build

bash 
cmake --build build/release -j$(nproc)

# Produced binaries
build/release/src/libmodelrepair.so   # shared library
build/release/cli/model-repair        # CLI tool
build/release/gui/model-repair-gui    # Qt desktop app

4 — Run the tests

bash 
ctest --preset release --output-on-failure

How it works

Ordered repair pipeline

Each step is a hard precondition for the next. Running them out of order produces incorrect results — for example, normal orientation only works reliably on manifold meshes.

1

Merge duplicate vertices

Converts the raw polygon soup (how STL files are stored) into a proper half-edge mesh. Calls PMP::repair_polygon_soup which also removes degenerate and duplicate polygons before rebuilding the mesh.

2

Remove degenerate triangles

Eliminates zero-area faces using PMP::remove_degenerate_faces. These corrupt every subsequent topological operation.

3

Fix non-manifold geometry

Splits vertices where the face fan is not disk-topology using PMP::duplicate_non_manifold_vertices. Ensures every edge is shared by exactly two faces.

4

Fix face normals

Orients each connected component so all normals point outward using PMP::orient. Skips the closed-mesh check on open surfaces to avoid spurious CGAL warnings.

5

Fill holes

Patches open boundary cycles using PMP::triangulate_and_refine_hole (smooth) or PMP::triangulate_hole (flat). Holes are collected via full boundary-cycle traversal; a --max-hole-edges limit skips large intentional openings.

6

Remove self-intersections slow

Converts to EPECK (exact constructions kernel) for robustness, calls PMP::experimental::remove_self_intersections, then converts back to EPICK. Disable with --no-remove-self-intersections for fast passes.

CLI reference

Command-line tool

All repair steps are enabled by default. Disable any step individually with a --no-* flag. Format is inferred from the file extension.

usage 
model-repair INPUT OUTPUT [OPTIONS]
examples 
# Basic repair — all steps, binary STL output
model-repair broken.stl repaired.stl

# Verbose per-step progress with JSON report
model-repair broken.stl repaired.stl --verbose --report report.json

# Convert STL → 3MF while repairing
model-repair broken.stl repaired.3mf

# Skip the slow self-intersection pass
model-repair broken.stl repaired.stl --no-remove-self-intersections

# Only fill holes smaller than 20 edges
model-repair broken.stl repaired.stl --max-hole-edges 20

Architecture

Library + two frontends

All repair logic lives in libmodelrepair.so. Both frontends link it dynamically — satisfying CGAL's LGPL terms without imposing license constraints on the frontends.

libmodelrepair.so (src/ · include/modelrepair/) ├── Mesh CGAL Surface_mesh<EPICK::Point_3> wrapper ├── RepairPipeline run(Mesh&) → RepairReport ├── RepairOptions plain struct — one bool/value per step ├── RepairReport vector<StepReport> + mesh stats └── io/ MeshIO · StlIO · ObjIO · ThreeMFIO model-repair (cli/) CLI11 + spdlog model-repair-gui (gui/) Qt 6 Widgets + Concurrent + OpenGL ├── MainWindow drag-drop, options checkboxes, Save As ├── RepairWorker QObject on QThread — emits before+after mesh ├── ReportView QTreeWidget rendering RepairReport ├── PreviewWindow side-by-side Before/After 3D window └── MeshViewWidget QOpenGLWidget — Phong shading, arcball camera Dependencies (system packages) ├── CGAL 6.x Polygon Mesh Processing — all repair ops ├── Eigen3 required by CGAL ├── Boost / GMP / MPFR required by CGAL └── Qt 6 Widgets + Concurrent + OpenGL GUI only Dependencies (auto-fetched via FetchContent) ├── lib3mf v2.4.1 3MF format — Apache-2.0 ├── CLI11 v2.4.2 argument parsing — MIT ├── spdlog v1.15.3 logging — MIT └── Catch2 v3.8.1 tests — BSL-1.0

Kernel strategy

Steps 1–5 use EPICK (Exact Predicates, Inexact Constructions) — fast floating-point coordinates with exact predicate evaluation. Step 6 converts to EPECK (Exact Predicates, Exact Constructions) via CGAL::copy_face_graph for robust self-intersection removal, then converts back. This is transparent to callers.

Bambu Studio integration

wrap-repair.sh 
#!/bin/bash
# Repair a model and open it in Bambu Studio
INPUT="$1"
REPAIRED="${INPUT%.stl}_repaired.stl"
model-repair "$INPUT" "$REPAIRED" --verbose && bambu-studio "$REPAIRED"
